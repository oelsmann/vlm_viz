This is a showcase website intended to motivate community efforts across the vertical land motion (VLM) community to make VLM data more accessible, transparent, interpretable, and applicable.

Static deck.gl GlobeView visualization of GPS, GIA, InSAR, tide-gauge, hybrid InSAR/GNSS vertical land motion datasets, and optional external context layers.

This prototype is intended to evolve into a community effort where researchers and data providers from the VLM community are welcome to co-develop the code, metadata, and documentation through GitHub. Authors and custodians of original datasets will be contacted and invited to contribute, correct metadata, improve provenance, and help shape the platform into a joint project.

## Idea

- The entire website can be transparently rebuilt from a single Python generator script, making the processing logic inspectable and easy to improve.
- All displayed scientific layers are fetched or rebuilt from public source records, repositories, or archives with persistent links, DOIs, and citation targets where available.
- The workflow is fully static and reproducible: preprocessing happens at build time, while the published website runs as plain HTML, JavaScript, CSS, and static data payloads.
- Static hosting keeps the platform easy to mirror, archive, review, and deploy on GitHub Pages or similar services without a backend server.
- Dataset metadata, provenance, source URLs, and classification notes are stored as structured files so they can be reviewed, corrected, and extended by the community.
- Large raw datasets can stay outside Git history while compact render payloads and scripts preserve a reproducible path back to the original sources.

## Files

- `generate_midas_globe.py` parses cached/source data and generates the map and catalogue HTML files.
- `vlm_dataset_sources.py` owns dataset URLs, downloads, local caches, extraction, and legacy data-path migration.
- `index.html` is generated by the script and can be hosted as a static page.
- `catalogue.html` is generated by the script and contains a searchable/filterable dataset metadata table.
- `datasets/gnss_blewitt_2018/` contains the raw cached MIDAS source file.
- `datasets/gia_caron_2020/` contains the raw cached JPL VESL GIA grid.
- `datasets/insar_ohenhen_2025/` contains the Zenodo InSAR delta VLM ZIP and extracted GeoTIFF grids.
- `datasets/insar_gnss_hamling_2022/` contains the GNS New Zealand Envisat InSAR/GNSS VLM archive, extracted files, and metadata XML.
- `datasets/tide_gauge_dangendorf_2026/` contains the Zenodo CSL-TG MATLAB file and a processed JSON cache of fitted tide-gauge trends.
- `external_datasets/ghsl_schiavina_2025/` contains the GHSL GHS-WUP-POP 2025 population ZIP and extracted raster files.
- `external_datasets/ghsl_schiavina_2025/ghsl_population_payload.js` contains the sampled population render payload and is loaded only when the population layer is toggled on.
- `dataset_attributes/` contains per-dataset metadata used by the sidebar info buttons and catalogue:
  `gnss_blewitt_2018.json`, `gia_caron_2020.json`, `insar_ohenhen_2025.json`, `insar_gnss_hamling_2022.json`, and `tide_gauge_dangendorf_2026.json`.
- `external_dataset_attributes/` contains minimal external-context metadata, currently `ghsl_schiavina_2025.json`.

## Generate or refresh

Create the conda environment:

```bash
conda env create -f environment.yml
```

```bash
conda run -n vlm_viz python generate_midas_globe.py --force-refresh
```

To regenerate from the local cache:

```bash
conda run -n vlm_viz python generate_midas_globe.py
```

The CSL-TG MATLAB v7.3 parser needs `h5py` and `numpy` unless the processed JSON cache already exists. These are included in `environment.yml`.

The generator uses:

- MIDAS data: <https://geodesy.unr.edu/gps_timeseries/IGS20/midas/midas.IGS.txt>
- MIDAS column reference: <https://geodesy.unr.edu/velocities/midas.readme.txt>
- GIA grid: <https://vesl.jpl.nasa.gov/solid-earth/gia/downloads/GIA_maps_Caron_Ivins_2019>
- InSAR delta grids: <https://zenodo.org/records/15015923/files/gridVLM.zip?download=1>
- Associated article: <https://www.nature.com/articles/s41586-025-09928-6>
- GNS New Zealand InSAR/GNSS VLM metadata: <https://data.gns.cri.nz/metadata/srv/eng/catalog.search#/metadata/fdbb8847-c882-4324-ae48-ca7ed9b7433b>
- GNS New Zealand InSAR/GNSS VLM archive: <https://data.gns.cri.nz/metadata/srv/api/records/fdbb8847-c882-4324-ae48-ca7ed9b7433b/attachments/NZ_InSAR_GPS%20(6).zip>
- Associated New Zealand VLM article: <https://agupubs.onlinelibrary.wiley.com/doi/full/10.1029/2021GL096465>
- CSL-TG tide-gauge VLM record: <https://zenodo.org/records/18777050>
- CSL-TG MATLAB file: <https://zenodo.org/records/18777050/files/CSL-TG.mat?download=1>
- GHSL GHS-WUP-POP 2025 population raster: <https://jeodpp.jrc.ec.europa.eu/ftp/jrc-opendata/GHSL/GHS_WUP_POP_GLOBE_R2025A/GHS_WUP_POP_E2025_GLOBE_R2025A_54009_1000/V1-0/GHS_WUP_POP_E2025_GLOBE_R2025A_54009_1000_V1_0.zip>

The same diverging `+/- mm/yr` color scale is used for GPS UP velocity, GIA vertical land motion, InSAR delta vertical land motion, GNS New Zealand coastal VLM, and CSL-TG tide-gauge VLM trends. The InSAR GeoTIFF pixel values are converted to `mm/yr` with a factor of `10`, because the stored values are inferred to be in `cm/yr`.

The GHSL population layer is an external context layer, not a VLM dataset. It is off by default, uses a separate transparent sequential color texture, and its attributes are stored separately from the VLM catalogue records.

Clicking a GPS station opens a station plot modal. The static page loads the UNR-rendered PNG time-series plot for that station only after the click, so it avoids fetching thousands of `.tenv3` tables and does not need browser-readable CORS for the text files. The modal also shows the MIDAS UP trend and uncertainty from the embedded velocity table.

Clicking a CSL-TG tide-gauge square opens an embedded annual MDadj3 residual time-series plot with the fitted linear trend and uncertainty. The popup also shows the 10 closest MIDAS GNSS trends within 100 km when available; solid line segments cover the station period and dashed lower-opacity segments show extrapolation over the tide-gauge period.

To refresh the GIA grid too:

```bash
conda run -n vlm_viz python generate_midas_globe.py --force-refresh --refresh-gia --refresh-insar --refresh-gns --refresh-tide-gauge --refresh-external
```

## View locally

Open `index.html` or `catalogue.html` directly in a browser, or serve the folder with any static file server:

```bash
conda run -n vlm_viz python -m http.server 8000
```

Then visit <http://localhost:8000/>.

## GitHub Pages

Commit `index.html`, `catalogue.html`, `dataset_attributes/`, `external_dataset_attributes/`, `datasets/`, `external_datasets/`, `generate_midas_globe.py`, `vlm_dataset_sources.py`, and this README. Enable GitHub Pages for the branch and folder containing `index.html`.

To keep the page fresh, rerun:

```bash
conda run -n vlm_viz python generate_midas_globe.py --force-refresh
```

Then commit the updated `index.html`, `catalogue.html`, `dataset_attributes/`, `external_dataset_attributes/`, `datasets/`, and `external_datasets/`. The MIDAS README says the velocity files are updated weekly by the Nevada Geodetic Laboratory.
